1. How does javascript work?

Ans: Javascript is regarded as a client-side scripting language which runs inside an internal browser. When some javascript code are placed inside the web pages and when the browser loads the page, the browser has built in interpreter that reads the javascript code that it finds in the page and run's it.JavaScript is most often run on webpages inside the browser, but it can also be run server-side. Java needs to be compiled before it is run.The source code is passed through a program called a compiler, which translates it into bytecode that the machine understands and can execute. In contrast, JavaScript has no compilation step. Instead, an interpreter in the browser reads over the JavaScript code, interprets each line, and runs it.

2: What are the difference between Local Storage and Session Storage?

LocalStorage
SessionStorage
1. Local Storage use for long term use.
1. Session Storage use for temporary use
2. Local storage data will not expire.
2. Session Storage is lost the moment the browser is closed
3. Data stored in the local storage is also available after the window has been closed.
3. When the browser window is closed, the session storage associated with that window is deleted.
